# 车辆动力学与碰撞逻辑规则

## 目的

确保生成的场景在车辆运动、坡道行为和碰撞机制上物理合理、逻辑一致。

---

## 一、坡道场景规则

### 1.1 重力方向定义

- **上坡道路**：车辆前向行驶时需要克服重力分量
- **下坡道路**：车辆前向行驶时重力分量推动车辆前进

### 1.2 溜车场景物理机制

| 场景类型 | 车辆状态 | 危险运动方向 | 必要条件 |
|---------|---------|-------------|---------|
| 下坡前溜 | 启动起步/泊车/驻车 | 向前（下坡方向） | 制动力不足，重力分量 > 制动力 |
| 上坡后溜 | 启动起步/泊车/驻车 | 向后（下坡方向） | 制动力不足，重力分量 > 制动力 |
| 平路溜车 | 启动起步/泊车/驻车 | 前向或后向 | 制动力为0，外力推动或未挂P挡 |

**关键规则**：
- 溜车方向**必须**与坡道下降方向一致
- `道路条件`必须选择`上坡道路`或`下坡道路`
- `车辆状态`应为`启动起步`、`泊车`、`驻车`或`停车未熄火`
- `车速(km/h)`应为`(0，10]`或`静止状态`
- `附加条件`必须说明坡度和制动状态

### 1.3 溜车危害事件填写模板

```
下坡前溜：<风险对象>因车辆在<坡度>下坡道路上非预期向前滑动，发生<碰撞/跌落>受到伤害风险
上坡后溜：<风险对象>因车辆在<坡度>上坡道路上非预期向后滑动，发生<碰撞/跌落>受到伤害风险
```

**验证检查清单**：
- [ ] `道路条件` = `下坡道路` 时，危险方向是向前
- [ ] `道路条件` = `上坡道路` 时，危险方向是向后
- [ ] `附加条件`说明了坡度（如5%坡度、陡坡）
- [ ] 风险对象位于溜车方向的前方路径上

## 二、行人风险场景规则

### 2.1 行人位置与车辆危险路径一致性

| 车辆状态 | 危险方向 | 行人应位于 | 禁止搭配 |
|---------|---------|-----------|---------|
| 前向加速/匀速 | 车辆前方 | 车辆行进路线前方 | 行人在车辆后方时称碰撞风险 |
| 后向行驶 | 车辆后方 | 车辆后方路径上 | 行人在车辆前方时称碰撞风险 |
| 上坡后溜 | 向后 | 坡道下方后方区域 | 行人在坡道上方时称碰撞风险 |
| 下坡前溜 | 向前 | 坡道下方前方区域 | 行人在坡道上方时称碰撞风险 |
| 转弯 | 转弯方向 | 转弯路径内侧/外侧 | 行人在转弯反方向时称碰撞风险 |

**关键规则**：
- 风险对象（行人）**必须**位于车辆实际运动或潜在运动的危险路径上
- `特殊要素`应选择`行人数量(少)`或`行人数量(多)`
- `附加条件`必须说明行人相对位置（如"行人位于车辆前方5米处"）

### 2.2 行人危害事件填写模板

```
前向碰撞行人：<风险对象>因车辆在<场景条件下>非预期前向运动，与前方<位置>的行人发生碰撞受到伤害风险
倒车碰撞行人：<风险对象>因车辆在<场景条件下>非预期后向运动，与后方<位置>的行人发生碰撞受到伤害风险
```

**验证检查清单**：
- [ ] 车辆运动方向与行人位置一致
- [ ] `附加条件`说明了行人相对位置和距离
- [ ] 考虑了行人是否有足够的反应时间和避让空间

## 三、追尾与被追尾场景规则

### 3.1 追尾场景（本车追尾前车）

**触发条件**：
- 本车非预期减速失效或减速不足
- 本车非预期加速
- 前车紧急制动，本车制动距离不足

**必要字段组合**：
```
"车辆状态": "前向加速" / "前向匀速" / "前向减速/刹车"
"特殊要素": "与前车距离较近"
"附加条件": "前车突然制动，本车制动距离不足"
```

**危害事件模板**：
```
本车追尾：本车乘员和前车乘员因本车非预期<加速/制动失效>，与前车发生追尾碰撞受到伤害风险
```

### 3.2 被追尾场景（后车追尾本车）

**触发条件**：
- 本车非预期急减速（后车反应时间不足）
- 本车在行进中突然失去动力/停车
- 本车非预期急减速导致后车制动距离不足

**必要字段组合**：
```
"车辆状态": "前向减速/刹车" / "前向滑行" / "前向爬行(缓慢前进)"
"特殊要素": "与后车距离较近"
"附加条件": "后车跟车距离不足，本车非预期减速/停车"
```

**危害事件模板**：
```
被后车追尾：本车乘员和后车乘员因本车非预期<急减速/停车>，导致后车制动距离不足发生追尾碰撞受到伤害风险
```

### 3.3 连环追尾场景

**触发条件**：
- 本车非预期急减速，前后车距均不足
- 高速/拥堵路况下制动失效

**必要字段组合**：
```
"车辆状态": "前向减速/刹车"
"特殊要素": "与前车距离较近" + "与后车距离较近"（在附加条件中说明）
"附加条件": "高速跟车，前后车距均不足，本车非预期减速"
```

### 3.4 追尾场景验证检查清单

- [ ] 明确是追尾还是被追尾（二者的故障机制不同）
- [ ] 追尾：`特殊要素` = `与前车距离较近`
- [ ] 被追尾：`附加条件`说明`与后车距离较近`
- [ ] `车辆状态`与碰撞机制一致（加速追尾 vs 减速被追尾）
- [ ] 考虑了车速对碰撞能量的影响（高速追尾风险更高）

## 四、场景一致性验证总表

生成场景后，必须回答以下问题：

| 验证项 | 问题 | 示例错误 |
|-------|------|---------|
| 坡道-运动一致性 | 坡道方向与溜车方向是否一致？ | 上坡道路上"向前溜车" |
| 风险对象位置 | 风险对象是否在危险路径上？ | 车辆前向运动时"后方行人碰撞风险" |
| 速度-状态一致性 | 车速与车辆状态是否匹配？ | `驻车`状态下车速`(90，120]` |
| 制动-后果一致性 | 制动失效/制动过小的后果是否合理？ | 制动过小导致"车辆非预期加速" |
| 碰撞方向一致性 | 碰撞描述与车辆运动方向是否一致？ | 前向行驶时"被后车追尾"但车辆状态为加速 |
| 驾驶员在环 | 驾驶员能否感知和控制？ | 驾驶员在车上但不可控性高时未解释原因 |

## 五、常见错误模式与修正

### 错误模式1：坡道与溜车方向矛盾

```
❌ 错误：
"道路条件": "上坡道路"
"车辆状态": "启动起步"
"危害事件": "车辆非预期向前滑动，与前方行人碰撞"

✅ 修正：
"道路条件": "上坡道路"
"车辆状态": "启动起步"
"附加条件": "5%上坡，起步时制动力不足"
"危害事件": "车辆非预期向后滑动，与坡道下方后方行人碰撞"
```

### 错误模式2：行人位置与运动方向矛盾

```
❌ 错误：
"车辆状态": "前向加速"
"特殊要素": "行人数量(少)"
"危害事件": "与后方行人碰撞"

✅ 修正：
"车辆状态": "前向加速"
"特殊要素": "行人数量(少)"
"附加条件": "行人位于车辆前方行进路线"
"危害事件": "与前方行人碰撞"
```

### 错误模式3：追尾方向与车辆状态矛盾

```
❌ 错误：
"车辆状态": "前向加速"
"特殊要素": "与前车距离较近"
"危害事件": "被后车追尾"

✅ 修正：
"车辆状态": "前向减速/刹车"
"特殊要素": "与前车距离较近"
"附加条件": "与后车距离也较近"
"危害事件": "本车非预期急减速，导致后车制动距离不足被追尾"
```

## 六、在场景推理中记录物理机制

每个涉及车辆动力学的场景，`scenario_reasoning.危害事件推理`必须包含：

```json
"危害事件推理": "故障导致<制动/驱动/转向>系统输出异常，车辆在<坡度/平路>条件下受<重力/惯性>作用发生<实际运动方向>运动，风险对象位于<相对位置>处于危险路径上，导致<碰撞/跌落>风险"
```

**示例**：
```
"危害事件推理": "驻车制动卡滞导致制动力不足，车辆在5%下坡道路上受重力分量作用向前滑动，行人位于坡道下方前方5米处处于危险路径上，导致车辆与行人碰撞风险"
```
