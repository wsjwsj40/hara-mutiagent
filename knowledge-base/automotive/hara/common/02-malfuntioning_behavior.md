# HARA 故障描述生成
本文件用于指导 HARA 分析中的**故障描述生成**。故障描述阶段的目标是：基于 Item Definition 或系统功能清单，识别每个功能可能出现的异常行为，即 ISO 26262 语境下的 **malfunctioning behavior**。
在 ISO 26262 中，HARA 的目标是识别并分类由 item 的 malfunctioning behavior 触发的 hazard，并据此形成 safety goal；因此，故障描述是后续风险识别、场景生成、SEC 计算、ASIL 判定和安全目标生成的输入。

## 1. 阶段边界
故障描述阶段只描述：某个车辆级、系统级或功能级行为相对于设计意图发生了怎样的异常。
典型输出包括：
| 字段                  | 说明          |
| -------------------- | ----------- |
| Function ID          | 功能编号        |
| Function Name        | 功能名称        |
| Malfunction Type     | 故障类型        |
| Fault Description    | 故障描述        |

## 2. 核心定义
### 2.1 故障行为
**故障行为** 可理解为：item 或功能相对于设计意图发生的失效行为或非预期行为。
ISO 26262 公开术语中将 malfunctioning behaviour 描述为 item 相对于设计意图的 failure 或 unintended behaviour。

### 2.2 故障描述
在本知识库中，**故障描述**定义为：对某一功能的异常输出、异常状态、异常动作或异常时序的结构化表述。
推荐格式：
```text
在[功能适用前提]下，[功能/系统]产生[异常行为]。
```
示例：
```text
车辆动态制动时,EPB卡滞无法制动。
```

## 3. 故障类型
### 3.1 故障类型分类
本知识库使用以下故障提示词作为故障描述生成的主分类：

| 故障类型     | 英文参考                                 | 主要适用对象                |
| -------- | ------------------------------------ | ------------------- |
| 功能丧失     | Loss of function                     | 功能、信号、请求、状态、信息      |
| 过大       | More than intended                  | 输出值、动作强度、持续时间、频率   |
| 过早       | Too early                           | 动作时机、触发时机、状态切换时机   |
| 过小       | Less than intended                  | 输出值、动作强度、持续时间、频率   |
| 过晚       | Too late                            | 动作时机、触发时机、状态切换时机   |
| 非预期激活    | Unintended activation                | 功能、执行器、警告、控制请求      |
| 卡滞       | Stuck / stuck-at                     | 输出、状态、模式、目标识别、控制量   |
| 方向错误     | Wrong direction                      | 转向、加减速、挡位、目标方向、路径方向、互斥状态/动作 |

### 3.2 故障类型解释
#### 功能丧失
定义：功能在应该提供服务、输出、控制、判断或信息时，没有提供。
示例：
- 制动请求没有产生制动力；
- 驻车请求没有完成夹紧；
- 报警条件满足但没有报警。

#### 过大
定义：功能输出的幅值、持续时间、频率、动作速度或作用强度超过设计意图。
示例：
- 转向角度过大；
- 制动力、驱动力、扭矩、夹紧力超过合理范围；
- 力、扭矩、电流、压力、温度等幅值可能过大。
不适用场景：
- 由驾驶员主动请求且不存在独立提前触发机理的功能；
- 幅值过大不形成安全相关车辆级影响的功能。

#### 过早
定义：功能动作、触发、切换或释放/拉起/转向/制动等发生时间早于设计意图。
示例：
- ADAS 还未达到转向触发条件时提前转向；
- 自动驻车功能在驻车条件未满足时提前拉起；
- 自动释放功能在起步条件未满足时提前释放。
不适用场景：
- 功能完全由驾驶员即时主动请求触发，且没有自动提前触发机理；
- “提前”只是驾驶员提前操作，而不是系统相对设计意图提前执行。

#### 过小
定义：功能输出的幅值、持续时间、频率、动作速度或作用强度低于设计意图。
示例：
- 制动力不足；
- 扭矩输出不足；
- 转向角度不足；
- 驻车夹紧力不足。

#### 过晚
定义：功能动作、触发、切换、拉起、释放、转向、制动或告警发生时间晚于设计意图。
示例：
- 拉起过晚；
- 报警过晚。

#### 非预期激活
定义：功能在没有触发条件、没有驾驶员请求或不满足运行条件时被激活。
示例：
- 未请求制动时发生制动；
- 未请求释放时释放；
- 不满足条件时自动启动。

#### 卡滞
定义：功能输出、状态、模式、识别结果或控制量保持在某一固定值或固定状态，无法根据输入或环境变化更新，导致功能无法完成或无法退出。
示例：
- 无法夹紧；
- 无法释放；
- 保持在错误状态。

#### 方向错误
定义：功能输出方向、控制方向、判断方向或运动意图与设计意图相反或偏离。
判断方法：
- 先抽取"触发/请求/条件 -> 预期动作或目标状态"的配对关系；
- 再检查同一执行对象、控制量或状态机中是否存在互斥对立面；
- 即使当前子功能只描述其中一个方向，只要系统边界中存在同一对象的相反动作或状态，也应判断"请求 A 却执行相反 B"是否成立；
- 若只有大小/强弱变化而没有正反、互斥方向或相反状态，则通常不适用。
示例：
- 请求拉起却释放；
- 请求左转却产生右转控制；
- 请求降扭却升扭。
