# HARA 安全目标生成

## 定义

Safety goal 是针对不可接受风险定义的顶层安全目标，应描述必须避免或控制的车辆级危险，而不是具体软件或硬件实现。

## 生成规则

- 面向整车级 hazard
- 覆盖对应 malfunction 的非 QM hazardous events
- 必须与故障内容一致
- 必须与危害事件和整车级危害一致
- 表达为"防止、避免、限制、确保"等安全目标语义
- 不写成底层技术需求

### 与故障类型的对应关系

| 故障类型 | 安全目标表达方向 |
|---------|----------------|
| 功能丧失/失效 | 防止该失效导致的车辆级危险 |
| 过小/不足 | 防止输出不足导致的车辆级危险 |
| 过早/过晚 | 防止错误时序导致的车辆级危险 |
| 过大/非预期 | 防止非预期或过度输出导致的车辆级危险 |
| 方向错误 | 防止错误方向动作导致的车辆级危险 |

### 推荐表达模板

```
防止{车辆级危险行为或危害}。
确保在{关键运行条件}下不发生{车辆级危险行为或危害}。
限制{非预期输出}以防止{车辆级危险}。
```

### 选择依据

- 优先选择最高 ASIL 场景
- 如果多个场景同级，选择更能代表该 MF 本质危险的场景
- 如果所有场景为 QM，仍可给出简短安全目标，但不要强行拔高到 ASIL 安全目标语气

### 错误示例

| 错误写法 | 原因 | 正确写法 |
|---------|------|---------|
| 防止非预期减速 | 故障是非预期纵向移动，不能写成非预期减速 | 防止非预期纵向移动 |
| 确保传感器数据正确 | 这是底层技术需求 | 防止因传感器错误导致的非预期运动 |
| 制动系统正常工作 | 这是功能需求，不是安全目标 | 防止制动丧失导致车辆无法减速 |
