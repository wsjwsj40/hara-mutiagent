# HARA FTTI（Fault Tolerant Time Interval）判断

## 定义

FTTI 是从故障发生到危险无法避免之间允许系统检测、诊断并达到安全状态的时间窗口。

## 判断依据

FTTI 判断应考虑：
- 危险演化速度
- 故障检测时间
- 系统执行安全反应的时间
- 驾驶员或道路使用者可用反应时间
- 当前场景的速度、距离、坡度、交通参与者和空间约束

## 判断框架

1. 先估计从 malfunction 出现到危险状态不可避免之间的时间窗口
2. 再扣除检测、诊断、仲裁、执行器动作和提示传播所需时间
3. 若依赖驾驶员介入，还应说明驾驶员是否在场、能否感知以及是否有可行动作

## 写作要求

- FTTI 不应任意给出
- 理由应说明工程判断依据，例如：
  - 车辆运动从异常到危险状态的时间
  - 执行器响应时间
  - 诊断周期
  - 场景中的可用反应窗口

## 示例表达

```
FTTI: 100ms
理由: 车辆在(60，90]km/h速度下，1秒内可行驶约17-25米，考虑制动系统响应时间约100-200ms，驾驶员反应时间约300-500ms，剩余可用时间窗口为100-300ms
```

```
FTTI: 500ms
理由: 泊车场景下速度低于10km/h，碰撞能量较低，考虑系统检测和执行时间，留有足够安全余量
```
