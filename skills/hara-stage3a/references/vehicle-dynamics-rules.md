# 车辆动力学与碰撞逻辑规则

## 目的

提供车辆运动、坡道行为和碰撞机制的物理约束与校验规则，确保生成的场景逻辑自洽。本文件不提供具体场景模板或字段组合，场景应由模型基于 MF 故障和功能运行域独立推理生成。

---

## 一、纵向移动的力源约束

### 1.1 必要力源

任何"非预期纵向移动/溜车/滑动"场景都必须存在实际纵向力来源：

- **坡道重力分量**：`道路条件` 必须选 `上坡道路` 或 `下坡道路`，`附加条件` 写明坡度和车辆朝向。
- **驱动或传动输出**：如停车未熄火且 D/R 挡蠕行、未挂 P 挡、驱动扭矩未释放等。
- **外部作用**：如被后车推动、牵引、装卸外力等。

冰雪路面、降雨、冰雹、玻璃起雾、夜间、前车距离近等条件不能单独产生纵向移动危害。它们只能作为附加影响因素，且只有确实改变故障到危害事件的链路时才标为相关。

### 1.2 重力方向定义

- **上坡道路**：车辆前向行驶时需要克服重力分量，溜车方向为向后。
- **下坡道路**：车辆前向行驶时重力分量推动车辆前进，溜车方向为向前。

### 1.3 溜车方向规则

- 溜车方向**必须**与坡道下降方向一致。
- 存在坡度、陡坡、上坡、下坡描述时，`道路条件`必须选择`上坡道路`或`下坡道路`，不能选择`直道`。
- `车辆状态`应为`启动起步`、`泊车`、`驻车`或`停车未熄火`。
- `车速(km/h)`应为`(0，10]`或`静止状态`。
- `附加条件`必须说明坡度/力源和制动状态。
- 平路溜车必须存在外力推动、未挂 P 挡、D/R 挡蠕行或其他驱动/传动输出等实际力源。

## 二、行人位置与危险路径一致性

### 2.1 一致性约束

- 风险对象（行人）**必须**位于车辆实际运动或潜在运动的危险路径上。
- 车辆前向运动时，行人在车辆行进路线前方；后向运动时，行人在后方路径上。
- 上坡后溜时，行人在坡道下方后方区域；下坡前溜时，行人在坡道下方前方区域。
- 转弯时，行人在转弯路径内侧或外侧。

### 2.2 字段填写规则

- `特殊要素`应选择`行人数量(少)`或`行人数量(多)`。
- `附加条件`必须说明行人相对位置和距离。

## 三、追尾与被追尾规则

### 3.1 方向与因果区分

- **追尾（本车撞前车）**：故障机制为本车非预期减速失效、减速不足或非预期加速。车辆状态应为前向运动。
- **被追尾（后车撞本车）**：故障机制为本车非预期急减速或突然失去动力/停车，导致后车制动距离不足。
- **连环追尾**：本车非预期急减速，前后车距均不足。

### 3.2 字段与方向一致性

- 追尾场景：`特殊要素`应体现与前车距离关系。
- 被追尾场景：`附加条件`应说明与后车距离关系。
- `车辆状态`与碰撞机制必须一致（加速追尾 vs 减速被追尾）。
- 高速场景碰撞能量更高，应在推理中体现。

## 四、场景一致性校验规则

生成场景后，必须逐项校验以下一致性：

| 验证项 | 校验要求 |
|-------|---------|
| 坡道-运动一致性 | 坡道方向与溜车方向必须一致 |
| 风险对象位置 | 风险对象必须在车辆危险路径上 |
| 速度-状态一致性 | 车速与车辆状态必须匹配（如驻车状态下不应有高车速） |
| 制动-后果一致性 | 制动失效/制动过小的后果必须与制动系统功能对应 |
| 碰撞方向一致性 | 碰撞描述与车辆运动方向必须一致 |
| 驾驶员在环 | 驾驶员在车上但不可控性高时，推理中必须解释原因 |
| 故障因果必要性 | 移除当前 MF 故障后，同一危害事件不应仍会发生 |
| 风险对象命名 | 危害事件中的受害对象必须与实际碰撞对象一致 |
| 条件相关性 | 标为相关的天气/特殊要素必须实际改变故障到危害事件的链路 |

### 常见不一致模式

- 坡道方向与溜车方向矛盾（如上坡道路上向前溜车）
- 行人位置与运动方向矛盾（如前向运动时与后方行人碰撞）
- 追尾方向与车辆状态矛盾（如加速状态被追尾）
- 低附着条件替代故障因果（如平路冰雪作为溜车唯一原因，缺少坡度或力源）
- 附加条件对象与危害事件对象不一致（如附加条件是前方停放车辆，危害事件写对向车辆乘员）

## 五、危害事件推理中的物理机制

涉及车辆动力学的场景，`scenario_reasoning.危害事件推理`必须包含：

- 故障导致的系统异常输出或异常状态
- 车辆在何种道路/力源条件下的实际运动方向
- 风险对象的相对位置及其与危险路径的关系
- 最终导致的碰撞/跌落等伤害类型
