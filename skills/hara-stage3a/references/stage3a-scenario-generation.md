# Stage 3A：场景与危害事件生成方法

本文件只定义 Stage3A 的语义生成方法，不定义 JSON schema 或上下文拆分流程。输出结构见 `json-contracts.md`，上下文生成见 `SKILL.md`。

## 输入边界

只使用当前 `stage3_context_<MF_ID>.json` 中的信息：

- `mf`：当前 Stage2 MF 行。
- `hazard_reasoning`：Stage2 的危害映射推理。
- `function_context` 或 `matched_functions`：由 Stage1 context 复用来的功能背景。
- `operating_domain_hints`：从功能背景提取的运行域提示。

不要读取完整 Stage0、完整 Stage2、风险评估知识库或安全目标知识库。

## 生成目标

- 每个 MF 生成 10-20 条真实可信场景。
- 只生成场景、危害事件、`scenario_reasoning` 和 `max_asil_planning`。
- 不生成 S/E/C、ASIL、安全目标或安全状态。
- 场景应覆盖可信最高风险路径，而不是机械枚举道路、天气和车速组合。

## 最大风险路径规划

生成场景前必须先写 `max_asil_planning`。规划时按三类风险驱动因子思考：

| 方向 | 要问的问题 |
|---|---|
| 可能提高 S | 故障如何造成更严重的人身伤害？风险对象是谁？伤害能量从哪里来？ |
| 可能提高 E | 该场景是否处于功能真实运行域？用户是否经常遇到？ |
| 可能提高 C | 异常是否难以及时感知？驾驶员或他人是否有足够时间和空间控制？ |

`max_asil_planning` 至少包含：

- 高风险因素分析
- 规划的场景原型
- 预期最大_ASIL（规划假设，不是最终评级）
- 规划理由

## 功能运行域

优先从 `function_context.function.detail_text`、`operating_domain_hints` 和 MF 描述中提取；只有旧数据 fallback 没有 `function_context` 时，才使用 `matched_functions.detail_text`：

- 触发源：驾驶员请求、自动触发、系统条件触发。
- 运行边界：车速、车辆状态、道路/环境前置条件。
- 预期输出：制动、转向、驱动、换挡、驻车、照明、警示、门锁、能量管理等。
- 排除条件：功能明确不工作或不应输出的状态。

请求相关故障通常必须发生在功能运行域内；只有“非预期激活、无请求输出、条件不满足仍输出”等故障可以考虑运行域外危险输出，并必须解释原因。

## 场景原型

优先生成这些原型：

- 可信最高风险原型：最可能形成最高风险的真实组合。
- 接近最高风险原型：改变一个关键因子，观察风险边界。
- 常见高暴露原型：高频但不一定最严重的运行场景。
- 可控性边界原型：驾驶员在环/不在环、能感知/难感知、有空间/无空间。
- 低风险或不适用边界原型：用于说明风险上限，数量要少。

同一核心风险机制最多保留 2 条，不要只替换天气、道路名或地标制造重复场景。

## 场景字段

六大场景字段必须逐字来自 `operation_scenarios.json` 对应枚举：

- `道路类型`
- `道路条件`
- `环境条件`
- `车辆状态`
- `车速(km/h)`
- `特殊要素`

库外细节只能写入 `附加条件`。对危害事件无影响的字段，在 `scenario_reasoning.场景条件相关性检查` 中写明“不涉及，理由”，验证工具可据此规范化为 `不涉及`。

## 危害事件

写 `危害事件` 前必须确认：

1. 车辆实际危险输出或异常状态。
2. 风险对象类型和位置。
3. 风险对象是否位于危险路径上。
4. 道路、车辆状态、车速和驾驶员是否在车上是否逻辑一致。

危害事件建议格式：

```text
<风险对象>在<场景条件下>因<车辆异常状态或危险输出>受到<伤害类型>风险
```

涉及坡道溜车、行人风险、追尾/被追尾时，必须同时遵循 `vehicle-dynamics-rules.md`。

## 推理记录

每个场景必须包含 `scenario_reasoning`：

- `场景规划理由`：说明覆盖了哪个风险原型。
- `危害事件推理`：说明故障、危害、场景如何推出危害事件。
- `场景条件相关性检查`：逐项说明六大场景字段相关或不涉及的理由。

## 筛除规则

生成前和生成后都要剔除：

- 不在功能运行域内的请求相关故障场景。
- 为提高风险硬拼的极端条件。
- 风险对象不在车辆危险路径上的场景。
- 危害事件和车辆状态、车速、道路条件互相矛盾的场景。
- 只替换枚举值但风险机制相同的重复场景。

验证命令以 `SKILL.md` 为准。
